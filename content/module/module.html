1.模块定义

 模块是定义应用的最主要方式 。模块包含了主要的应用代码。

 在AngularJS中，一个应用可以包含多个模块，每一个模块都包含了实现具体功能的代码。

2.模块的作用

 保持全局命名空间的清洁；

 编写测试代码更容易，并能保持其清洁，以便更容易找到互相隔离的功能；

 易于在不同应用间复用代码；

 使应用能够以任意顺序加载代码的各个部分。

3.定义模块的语法

angular.module('moduleName', [])

使用angular.module()方法来声明模块，这个方法能够接受两个参数:

  • 第一个是模块的名称;
  • 第二个是依赖列表，也就是可以被注入到模块中的对象列表。

4.获取模块的语法

angular.module('moduleName')

调用这个方法时如果只传递一个参数，就可以用它来引用模块

总结：不带中括号就是获取模块，带了就是定义一个新的模块！

提示：若当前模块依赖了另外一些模块，那么当前的模块可直接使用它所依赖的模块中定义的服务指令等功能，并在页面中可以使用依赖的模块中定义的控制器，即在当前的模块的控制器内在嵌套使用依赖的模块的控制器，当然这种嵌套也是在html页面中控制器嵌套使用方式。同时当前模块页面部分也可以使用依赖的模块中定义的过滤器；

5.注意事项

一个页面手动方式启动多个模块

AngularJS API之 bootstrap 启动

Angular JS的模块依赖

手动绑定（启动）模块的方法：bootstrap；

语法：angular.bootstrap(element, [modules], [config]);

参数说明：

  • element: 是绑定ng-app的dom元素；
  • modules：绑定的模块名字；
  • config：附加的配置；

注意：

  • 使用该方法前必须先定义模块，使用angular.module('moduleName', [])先定义模块；
  • 手动绑定模块，就不需要在页面中是用ng-app来绑定模块了；
  • AngularJS默认在一个html页面中只启动一个ng-app 模块，而且启动的模块是第一次出现ng-app声明的那个模块。

手动方式启动多个模块

 angular.element(document).ready(
   function (){
	angular.bootstrap(document.getElementById("id-1"),["module-1"]);
	angular.bootstrap(document.getElementById("id-2"), ["module-2"]);
	//…
   }
 );
